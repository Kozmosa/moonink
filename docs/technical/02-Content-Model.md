# 02 Content Model

## 1. Overview

MoonInk uses a dual-track content model supporting both knowledge-base pages and article-style writing. Classification is determined by file extension and frontmatter, not directory structure. This allows MoonInk to operate inside existing Markdown folders (including Obsidian vaults).

## 2. Content Types

### Article

Markup-origin content (primarily Markdown). Processed through the full DocFlow pipeline: `ParserAdapter → WikiLinker → RenderAdapter`.

An `.md` file is classified as an Article unless its frontmatter declares `type: page`.

### Page

Either an `.html` file (HTML passthrough) or an `.md` file with frontmatter `type: page`. Pages are processed through the `HtmlPassthroughParserAdapter` or the standard Markdown pipeline depending on source format.

## 3. Classification Rules

| File | Frontmatter `type` | Result |
|------|--------------------|--------|
| `.html` | any / absent | Page |
| `.md` | `page` | Page |
| `.md` | `article` or absent | Article |

Classification happens in two passes:
1. **Discovery pass** (`runtime/content_discovery.mbt`): `.html` → Page, `.md` → Article (default, without reading file contents).
2. **Frontmatter pass** (planned): re-classify `.md` files to Page when frontmatter `type: page` is present.

## 4. Frontmatter Schema

Core fields:

- `title` — display title
- `description` — short summary
- `type` — `"article"` or `"page"` (default: `"article"`)
- `date` — ISO 8601 date string (`YYYY-MM-DD`)
- `tags` — array of tag strings
- `draft` — boolean, default false

Extended fields go into `extra: Map[String, String]` for forward compatibility.

## 5. Identity And Routing

Page identity uses the source file path. Routes are generated by `core/route.mbt`:

- `pretty` style: `guides/intro.md` → `/guides/intro/`; `index.md` → `/`
- `direct` style: `guides/intro.md` → `/guides/intro.html`

WikiLink resolution (planned) will use the filename stem as a stable identifier.

## 6. Collection Semantics

- Article pages support chronological ordering (by `date`) and tag grouping.
- Page pages support hierarchical navigation (by path).
- Both tracks share rendering, frontmatter, and search infrastructure.
