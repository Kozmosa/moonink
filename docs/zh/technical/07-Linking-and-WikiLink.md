# 07 链接与 WikiLink

## 1. 动机

面向知识组织的发布场景，需要比普通 Markdown 路径引用更强的内部链接能力。

## 2. 支持的链接形式

- 相对 Markdown 链接；
- 锚点链接；
- `[[WikiLink]]` 语法。

## 3. 校验目标

MoonInk V1 应校验：

- 缺失的内部目标；
- 非法相对路径；
- 无法解析的 WikiLink；
- 在可能情况下校验无效锚点引用。

## 4. WikiLink 解析策略

WikiLink 应按以下优先级通过页面标识规则进行解析：

1. 精确匹配 `id`
2. 精确匹配标题
3. 匹配规范化 slug

如果解析后仍然存在歧义，构建过程应报告清晰诊断信息。

## 5. 未来演进

该子系统后续应支持反向链接与图谱型元数据，同时不改变核心页面标识模型。
