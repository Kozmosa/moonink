# 04 构建流水线

## 1. 流水线阶段

MoonInk V1 的构建流程将有意保持显式。

1. 加载配置
2. 校验配置
3. 发现文件
4. 解析 frontmatter
5. 解析 Markdown
6. 规范化路由与标识
7. 解析链接与 WikiLinks
8. 构建导航与站点模型
9. 渲染页面
10. 输出静态资源与搜索索引

## 2. 为什么分阶段处理很重要

这种分阶段方式带来三点好处：

- 更易调试；
- 测试职责边界更清晰；
- 为未来扩展点预留自然位置。

## 3. 中间输出

每个阶段都应产出显式的内部数据结构，而不是去原地修改共享的全局状态。

## 4. 失败策略

对于非法配置、不可恢复的路由冲突等结构性失败，构建应直接停止。对于断链等内容诊断问题，后续可以做成可配置策略，但 V1 至少要能够清晰报告。
