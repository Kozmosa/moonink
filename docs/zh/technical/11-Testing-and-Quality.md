# 11 测试与质量

## 1. V1 质量基线

MoonInk V1 将单元测试作为最低正式质量标准。

## 2. 优先测试目标

- 配置解析与校验
- frontmatter 解析
- Markdown 解析边界
- 路由生成
- 导航生成
- 内部链接校验
- WikiLink 解析
- 搜索索引生成

## 3. 推荐的 Fixture 策略

除单元测试外，项目还应保留一小组 fixture 站点用于回归检查，即使正式端到端测试会在后续阶段才引入。

## 4. 诊断质量

好的错误信息本身就是产品质量的一部分。错误输出应尽可能指出文件、失败规则以及可能的修复路径。

## 5. 后续质量扩展

后续阶段应逐步加入快照式输出校验、集成测试与更大规模的 fixture 构建。
