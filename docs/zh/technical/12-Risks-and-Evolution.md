# 12 风险与演进

## 1. 主要风险

### 生态成熟度

相较于拥有成熟解析与模板库的生态，MoonBit 可能需要更多自研基础设施。

### 范围膨胀

知识库型产品往往很早就会吸引反向链接、图谱视图与高级引用等功能诉求。

### 渲染复杂度

如果不加约束，主题系统与本地搜索都可能很快膨胀出复杂度。

### 多语言返工

如果页面标识仅依赖路径，那么未来引入多语言支持会变得困难。

## 2. 缓解原则

- 保持 V1 范围狭窄；
- 优先采用显式中间模型；
- 保持稳定的内容标识；
- 在内部模式未被验证前，延后生态 API 的开放。

## 3. 演进优先级

V1 之后最重要的演进方向是：

- 增量构建；
- 多语言支持；
- 部署集成；
- 正式的插件与主题扩展点。

## 4. 架构护栏

为了安全支持后续演进，MoonInk 应避免将解析、路由、渲染与搜索耦合成一个共享状态机。`docs/technical/00-Architecture.md` 中定义的边界应尽量保持稳定。
