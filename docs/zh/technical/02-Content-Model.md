# 02 内容模型

## 1. 概述

MoonInk V1 采用双轨内容模型，以同时支持稳定的知识层级与时间导向的写作流。

## 2. 内容类型

### 文档页

文档页属于层级化知识树，通常是概念结构的一部分，并出现在侧边导航中。

### 文章页

文章页属于流式集合，通常按日期排序，或按标签进行组织。

## 3. 通用元数据字段

- `title`：必需的展示标题
- `id`：推荐使用的稳定内容标识
- `date`：推荐用于文章排序与新鲜度信息
- `author`：可选的作者归属字段
- `tags`：可选的分类字段

## 4. 未来预留字段

MoonInk V1 可以预留但不强依赖以下字段：

- `summary`
- `draft`
- `order`
- `aliases`

## 5. 标识规则

页面标识不应只依赖文件路径。只要存在 `id`，就应优先用它进行稳定引用，尤其是在 WikiLink 解析与未来多语言演进场景下。

## 6. 集合语义

- 文档页按层级组织；
- 文章页按时间与标签组织；
- 两类内容尽可能共享渲染、元数据与搜索基础设施。
